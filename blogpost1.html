The Shuttle

A fedx simulator

Learning points

As this was to be a puzzle game I went through different iterations of puzzle ideas. One that I was set on having in the prototype was a Joystick controlled flight process with crashing and resetting objects. To achieve this I used a prebuit joystick from XR interaction tool kit This made the implimentation in unity quite easy, as all I had to do was read the values from the joy stick to move around the flight object.

I had the make two iterations of the project, as when I went to test some of the objects in the scene, the controlls would not work. This ended up being for the best, as it gave me time to rethink my original puzzles.






In the end I was happy with how I had implimented my designs, the quality of the game was not what I liked but this is something to improve in the future
Internal discussion section

During development of my first VR game I was learning how to apply many different processes. One of which was VR itself. For this project I wanted to develop a game that the user had to move objects around and search for differnt tools to use in creating a way to the finish. I beleive that the concepts that I introduced forced the player to think and engage with the enviroment, even if some of the ways the player interacted were not fully fleshed out.
